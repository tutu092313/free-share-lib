## 项目简介

**随心资料分享库** 是一款轻量级纯前端 Web 小程序，旨在解决大学生群体在跨专业、跨考试场景下资料分散、难以统一检索与分享的痛点。

设计目标：

- **通用性**：不绑定某一门课或某一场考试，**专业 + 领域**双维度可自由扩展
- **零门槛**：无需后端、无需注册即可使用全部基础功能
- **多格式**：PDF / Word / 图片三类最常见的学习资料均能在线预览
- **可扩展**：JSON 批量入库、Excel 表格粘入、文件批量投喂，支持后续对接真实后端

## 核心特性

### ✨ 三大业务模块

| 模块 | 能力 |
|---|---|
| **用户模块** | 注册 / 登录 / 登出 / 个人中心 / 编辑资料 / 体验账号一键登录 |
| **资料模块** | 上传 / 双维度分类 / 全文搜索 / 多格式在线预览 / 排序筛选 |
| **分享模块** | 评论 / 点赞 / 收藏 / 下载 / 删除自己上传的资料 |

### 🪄 通用投喂系统（亮点）

任何人都能把"任意专业 / 任意领域"的资料**批量**塞进资料库：

- **方式 1 · 粘贴 JSON**：支持单条 / 数组，兼容中英文字段名（`title` 或 `名称`、`major` 或 `专业`…）
- **方式 2 · 填表投喂**：从 Excel / 飞书 / 腾讯文档**直接复制粘贴**到表格中，一行一条
- **方式 3 · 文件批量**：把一堆 PDF / Word / 图片一次性拖进来，自动按扩展名归类
- **新专业 / 新领域自动入库**：字典里没有的，投喂一次后再上传就能在下拉里直接选

### 🔍 多格式在线预览

- **PDF** → PDF.js 渲染第一页缩略图
- **Word(.docx)** → mammoth.js 转 HTML
- **图片** → 直接展示原图

### 📱 响应式设计

- 桌面端：顶部横向导航 + 多列卡片网格
- 移动端：底部 4 栏 Tab + 单列 / 双列卡片自适应

## 快速开始

### 方式一：直接双击打开（最快）

进入 `site/` 目录，**双击 `index.html`** 即可在浏览器中打开。

> 所有数据保存在浏览器 `localStorage` 中，刷新不丢失。

### 方式二：本地起静态服务（推荐）

```bash
cd site
python -m http.server 8765
# 浏览器访问 http://localhost:8765/
```

> 方式二可避免某些浏览器对 `file://` 协议下 CDN 资源加载的限制。

### 方式三：Node.js 起服务（任选其一）

```bash
cd site
npx http-server -p 8765
```

## 功能详解

### 1. 用户模块

- **注册**：账号 + 密码 + 昵称 + 简介（简介可后续在个人中心补全）
- **登录 / 登出**：基于 `localStorage` 的本地会话
- **个人中心**：统计（上传数 / 收藏数 / 评论数）+ 三个 Tab（我的上传 / 我的收藏 / 我的评论）
- **体验账号**：不想注册时，登录弹窗点"**以体验账号登录**"即可使用 `demo / demo` 一键进入

### 2. 资料模块

#### 上传
- 支持 **PDF / Word(.docx) / 图片** 三种类型
- 单文件 ≤ 8MB（localStorage 容量上限约束）
- 拖拽 + 点击两种交互
- 上传时**自动生成缩略图**（PDF 抓首页 / 图片缩放 / Word 用类型图标）

#### 分类与检索
- **专业 × 领域**双维度筛选，可叠加使用
- 搜索：标题、描述、标签、专业名 全字段命中
- 排序：最新发布 / 最受欢迎（按点赞数）/ 下载最多

#### 预览
- PDF：使用 PDF.js 渲染第一页到 canvas
- Word：使用 mammoth.js 将 `.docx` 转 HTML 渲染
- 图片：直接 `<img>` 展示

### 3. 分享模块

- **评论**：登录后可发，显示昵称 + 时间，支持删除自己的评论
- **点赞 / 取消点赞**：实时计数，所有用户共享
- **收藏**：个人中心"我的收藏"可查看
- **下载**：示例数据下载为说明 txt（无真实文件）；用户上传的真实文件按原文件下载
- **删除**：仅资料上传者可删除

### 4. 投喂资料

#### 方式 1 · 粘贴 JSON
```json
[
  { "title": "考研英语·高频词 800", "major": "英语", "field": "kaoyan", "type": "pdf", "size": 320000, "desc": "近 5 年真题词频统计", "tags": ["考研", "词汇"] },
  { "title": "教资·小学综合素质 2024 真题", "major": "教育学", "field": "jiaozi", "type": "docx", "size": 540000, "tags": ["教资", "真题"] }
]
```
- 字段别名兼容：`title / 名称 / name`、`major / 专业`、`field / 领域`、`type / 类型`、`tags / 标签`
- 领域写中文名（如"教资"）或 id（如"jiaozi"）均可

#### 方式 2 · 填表投喂
- 表格字段：标题、专业、领域、类型、大小、标签、描述
- 从 Excel 复制多行 → 粘贴到网页表格 → 点击"表格入库"
- 每行可独立修改 / 删除

#### 方式 3 · 文件批量
- 选择**专业 + 领域**（支持填新值，自动入库）
- 拖拽多个文件 → 自动按 `.pdf` / `.docx` / 图片 归类
- 未填字段用文件名当标题

## 技术栈

| 类别 | 选型 | 理由 |
|---|---|---|
| **HTML** | HTML5 | 语义化标签 + 响应式 viewport |
| **CSS** | Tailwind CSS（CDN） | 实践作品免构建，工具类快速布局 |
| **JavaScript** | 原生 ES6+ | 零依赖、零打包，便于学习与演示 |

## 数据模型

### User（用户）
```js
{
  id: 'u_xxx',
  username: 'string',      // 账号
  password: 'string',      // 密码（明文存储，演示用；生产请 hash + 加盐）
  name: 'string',          // 昵称
  bio: 'string',           // 简介
  createdAt: timestamp
}
```

### Resource（资料）
```js
{
  id: 'r_xxx',
  title: 'string',         // 标题
  desc: 'string',          // 描述
  major: 'string',         // 专业（关联 Dict）
  field: 'string',         // 领域 id（关联 Dict）
  tags: ['string'],
  type: 'pdf' | 'docx' | 'image',
  size: number,            // 字节
  fileName: 'string',
  fileData: 'dataURL',     // 真实文件（用户上传时才有）
  thumb: 'dataURL',        // 缩略图
  uploaderId: 'u_xxx',
  uploaderName: 'string',  // 冗余字段，便于"社区贡献"显示
  createdAt: timestamp,
  downloads: number
}
```

### Comment（评论）
```js
{
  id, resourceId, userId, userName, text, createdAt
}
```

### Like（点赞） / Fav（收藏）
```js
// Like: { resourceId: [userId, ...] }
// Fav:  { userId: [resourceId, ...] }
```

### Dict（字典）
```js
// Majors: ['新闻学', '法学', ...]
// Fields: [{ id: 'kaoyan', name: '考研', icon: '🎓' }, ...]
```

## 扩展开发

实践二 / 三 可以在此基础上无缝升级：

### 1. 接入真实后端
- 后端选型建议：Node.js (Express / Koa) / Python (FastAPI / Django) / Go (Gin)
- 把 `Resource.add / Resource.all / Comment.add ...` 等方法替换为 `fetch('/api/...')`
- 保留 `App` 视图层与 `Dict` 字典结构不变

### 2. 文件存储
- 小文件：直接 base64 入库
- 大文件：OSS / MinIO / S3，URL 存数据库
- PDF 全文检索：pdf.js 提取文本 → Elasticsearch / Meilisearch

### 3. 用户体系增强
- 头像上传、关注 / 粉丝关系
- 邮箱 / 手机号验证、找回密码
- 第三方登录（微信 / GitHub / Google）

### 4. 内容治理
- 管理员审核、举报、敏感词过滤
- 评分系统（多维度评分 + 防刷）
- 付费 / 会员机制

### 5. 投喂系统增强
- 从外部 URL 批量拉取文件
- 批量导入 zip / tar.gz
- AI 自动打标（用大模型识别资料内容 → 推荐专业 + 领域）

## 常见问题

### Q1：清浏览器缓存后数据会丢吗？
会。所有数据存在 `localStorage`，**清缓存 = 清数据**。如果想要持久化，需要接入后端 + 数据库。

### Q2：能上传超过 8MB 的文件吗？
当前不能。`localStorage` 一般上限 5~10MB，且写入大量数据会卡顿。如有大文件需求，请接入对象存储。

### Q3：PDF / Word 预览只显示第一页吗？
PDF 当前只渲染第一页作为预览图（性能考虑）；Word 通过 mammoth 转 HTML 通常能完整展示。如需全 PDF 翻页，可改用 PDF.js 的多页渲染模式。

### Q4：示例资料里的"下载"为什么是 txt？
示例资料没有真实文件，下载时会生成一份说明 txt（包含资料标题、描述、专业、领域）。用户自己上传的文件会按原文件下载。

### Q5：投喂资料后立刻能在资料库看到吗？
能。投喂后会自动重新渲染资料库和"最近投喂"列表，刷新页面也在。
